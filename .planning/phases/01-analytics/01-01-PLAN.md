  <action>
SSH to Pollux and set up Umami. All commands run as root via `ssh pollux`.

**Step 1: Verify Apache mod_proxy is enabled**
```bash
ssh pollux "apache2ctl -M 2>/dev/null | grep proxy"
```
If `proxy_module` and `proxy_http_module` are NOT in the output:
```bash
ssh pollux "a2enmod proxy proxy_http && systemctl restart apache2"
```

**Step 2: Create /opt/umami/docker-compose.yml on the server**
```bash
ssh pollux "mkdir -p /opt/umami"
```

Write the file to /opt/umami/docker-compose.yml on the server with this exact content:
```yaml
services:
  umami:
    image: ghcr.io/umami-software/umami:latest
    ports:
      - "127.0.0.1:3000:3000"
    environment:
      DATABASE_URL: postgresql://umami:${POSTGRES_PASSWORD}@db:5432/umami
      APP_SECRET: ${APP_SECRET}
      DISABLE_TELEMETRY: 1
    depends_on:
      db:
        condition: service_healthy
    init: true
    restart: always
    healthcheck:
      test: ["CMD-SHELL", "curl http://localhost:3000/api/heartbeat"]
      interval: 5s
      timeout: 5s
      retries: 5

  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: umami
      POSTGRES_USER: umami
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - umami-db-data:/var/lib/postgresql/data
    restart: always
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U umami -d umami"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  umami-db-data:
```

CRITICAL: The port binding MUST be `127.0.0.1:3000:3000` — not `3000:3000`. Docker bypasses the system firewall with the short form, making port 3000 publicly accessible without authentication.

**Step 3: Generate secrets and write /opt/umami/.env**
```bash
ssh pollux "
  APP_SECRET=\$(openssl rand -base64 32)
  POSTGRES_PASSWORD=\$(openssl rand -hex 16)
  echo \"APP_SECRET=\${APP_SECRET}\" > /opt/umami/.env
  echo \"POSTGRES_PASSWORD=\${POSTGRES_PASSWORD}\" >> /opt/umami/.env
  chmod 600 /opt/umami/.env
  echo 'Secrets written'
"
```
`openssl rand -hex 16` produces hex-only output (no @, !, #, & chars that break PostgreSQL URL parsing).

**Step 4: Start the containers**
```bash
ssh pollux "cd /opt/umami && docker compose up -d"
```

**Step 5: Wait for health checks (Umami runs DB migrations on first start — allow 60-90 seconds)**
```bash
ssh pollux "cd /opt/umami && docker compose ps"
```
Both services must show "healthy" before proceeding to Task 2.

If not healthy after 2 minutes, check logs:
```bash
ssh pollux "cd /opt/umami && docker compose logs umami --tail=30"
```

  <how-to-verify>
**Part A: Configure KeyHelp Apache Reverse Proxy**

1. Log into the KeyHelp admin panel (typically at https://195.201.220.6:7443 or via your KeyHelp URL)
2. Navigate to Domains / Subdomains
3. Find or create the subdomain `analytics.bessersehen.la` under the bessersehen account
4. Open its **Apache Settings** (may be labeled "Additional directives" or "Apache configuration")
5. Add the following directives to BOTH the HTTP and HTTPS directive sections:

```apache
<IfModule mod_proxy.c>
    ProxyPass /.well-known/acme-challenge !
</IfModule>

Alias /.well-known/acme-challenge /home/keyhelp/www/.well-known/acme-challenge

ProxyPass / http://127.0.0.1:3000/
ProxyPassReverse / http://127.0.0.1:3000/
```

CRITICAL: The `ProxyPass /.well-known/acme-challenge !` line MUST come BEFORE the root `ProxyPass /` line. Order matters — the first matching rule wins.

6. Save the settings
7. In KeyHelp, request a Let's Encrypt SSL certificate for `analytics.bessersehen.la` if not already issued
8. Verify the proxy is working:
```bash
curl https://analytics.bessersehen.la/api/heartbeat
# Must return: {"ok":true}
```

**Part B: First Umami Login and Website Setup**

9. Open https://analytics.bessersehen.la in your browser
10. Log in with default credentials: username `admin`, password `umami`
11. IMMEDIATELY change the password: click your username (top right) → Profile → Change Password
12. Navigate to Settings → Websites → Add Website
13. Fill in:
    - Name: `Besser Sehen Landshut`
    - Domain: `bessersehen.la`
    - Leave "Enable share URL" unchecked
14. Click Save
15. For the new website entry, click Edit → Tracking Code (or the `</>` icon)
16. In the shown script tag, copy the UUID value from the `data-website-id` attribute
    - It looks like: `data-website-id="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"`
    - Copy only the UUID, without quotes
17. Disable self-tracking (so your own visits don't inflate stats):
    - Open browser developer console on analytics.bessersehen.la
    - Run: `localStorage.setItem('umami.disabled', 1);`
